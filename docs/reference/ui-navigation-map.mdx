# UI Navigation Map

A reference for where to find every feature in the QuickStack web interface.

## Application-level Tabs

| Tab | What you can do there |
|---|---|
| **Overview** | See deployment status, history, click **Deploy** |
| **Domains** | Add custom domains, quickstack.me URLs, view internal hostnames |
| **Environment** | Set environment variables |
| **Storage** | Add persistent volumes and file mounts |
| **Health Checks** | Configure liveness and readiness probes |
| **Networking** | Set internal networking rules |
| **Network Policies** | Restrict inter-app traffic |
| **Basic Auth** | Protect your app with username/password |
| **Webhooks** | Configure auto-deploy webhooks |
| **Logs** | View real-time and historical container logs |
| **Terminal** | Open a shell inside your running container |

## Project-level Navigation

| Section | What you can do there |
|---|---|
| **Apps** | Create and manage applications |
| **Databases** | Deploy databases from templates, manage DB connections |
| **Backups** | Configure S3 targets, schedule volume and database backups |

## Cluster Settings (Admin)

Navigate to **Settings** in the top navigation bar.

| Section | What you can do there |
|---|---|
| **QuickStack Server → Networking / Traefik** | Configure Let's Encrypt email, Traefik settings |
| **QuickStack Server → Registry** | Manage the built-in Docker registry |
| **Cluster → Nodes** | Add or remove cluster nodes |
| **Cluster → Monitoring** | View cluster resource usage |
| **Users & Groups** | Manage user accounts and permissions |
| **Two-Factor Authentication** | Enable/disable 2FA for your account |
| **Updates & Maintenance** | Update QuickStack and perform system maintenance |
