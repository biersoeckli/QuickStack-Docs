# S3 Targets

All QuickStack backup destinations are S3-compatible buckets. Configure at least one S3 target before setting up any backup schedule.

## Configuring an S3 target

1. Go to **Settings** → **S3 Targets**.
2. Click **Add S3 Target** and fill in:

| Field | Description |
|-------|-------------|
| **Name** | A label for this target (anything descriptive) |
| **Bucket Name** | Your S3 bucket name |
| **Endpoint** | S3-compatible endpoint URL (e.g. `s3.amazonaws.com`, `s3.eu-central-1.amazonaws.com`, your MinIO URL) |
| **Region** | Bucket region (e.g. `eu-central-1`) |
| **Access Key ID** | IAM/service account key with bucket permissions |
| **Secret Key** | Corresponding secret |

3. Click **Save**.

> **danger Credentials**
> Store Access Key and Secret Key securely. Rotate credentials regularly. The IAM policy needs: `s3:PutObject`, `s3:GetObject`, `s3:DeleteObject`, `s3:ListBucket`.

## Compatible providers

Any S3-compatible object storage works, including:
- AWS S3
- Cloudflare R2
- Backblaze B2
- MinIO (self-hosted)
- Hetzner Object Storage
- DigitalOcean Spaces
